[CMD Tutorial 3⁄4] Path Environment Variables

Thu May 09, 2019 by Super Cat in  Computer  Windows, Command

環境變數

Windows Command Prompt 也有變數的觀念，命令在執行的時候，可以讀取特定變數，然後就像執行命令時帶參數一樣，以不同的模式執行。 例如你可以開發一個aaa程式，執行時需要兩個參數，也可以使用ENV_AAA環境變數，

d:\> aaa c:\aaa d:\bbb

以上例子，兩個參數都是路徑，如果執行很多次會打字打個很累，正好aaa的開發人員寫成也可使用ENV_AAA 環境變數，所以下列方式也可以

d:\> set ENV_AAA="c:\aaa d:\bbb"

環境變數設定完成後，執行執行 aaa 就可以了，不需要打參數了。 

d:\> aaa

蠻多命令，都將不常變動的參數規劃放在環境變數，省下頻繁地打一堆字。 執行一份CMD的時候，作業系統會將全部的環境變數複製一份，這些環境變數專屬於該份CMD ，結束時會自動清掉。也就是說，如果同時執行兩份CMD，Windows 是複製出兩份系統的環境變數給這兩個CMD，兩邊各改了什麼，也不會影響對方，結束時也不會影響Windows系統的環境變數。 如果不小心改壞了什麼環境變數，可以關掉CMD，重新執行一次即可。

SET

想要看總共有哪些環境變數，可用set不帶任何參數

d:\> set

set 加上參數，就是設定環境變數了，語法為 set =

d:\> set ENV_AAA="c:\aaa d:\bbb"

ECHO

set一執行，舊式列出全部環境變數，如果只是想要看一個變數，可以用 echo 來顯示訊息。 C:\Users\Eric>set ENV_AAA=“c:\aaa d:\bbb”

C:\Users\Eric>echo %ENV_AAA%
"c:\aaa d:\bbb"

echo 也能印普通訊息，例如印個經典的 “Hello, World!”

C:\Users\Eric>echo Hello, World!
Hello, World!

PATH

當然，有一些名稱，被保留給系統使用，例如 PATH ，就是代表系統搜尋外部命令的路徑順序，每個路徑都必須以分號隔開。這個PATH環境變數因為太常用了，所以Microsoft乾脆開發一個PATH命令，專門負責顯示及修改PATH環境變數。

d:\> path

執行結果如下:

d:\>path
PATH=C:\Windows\system32;C:\Windows;C:\Windows\System32\Wbem;C:\Windows\System32\WindowsPowerShell\v1.0\;C:\Windows\System32\OpenSSH\;C:\Git\cmd;C:\Users\Eric\AppData\Local\Microsoft\WindowsApps;

上面Path變數的內容，其意義為 - 假設使用者想要執行 aaa.exe 執行檔，CMD 先看目前所在路徑，如果該檔案位於目前所在路徑，就載入記憶體執行，如果不在，則接著嚐試在 c:\windows\system32路徑找找看有無 aaa.exe 檔，如果還是沒有，則嚐試 c:\Windows，以此類推。 Path命令如果後面有參數，該參數會是覆蓋舊的值。所以，假設想要新增 “C:\Python\Scripts\;C:\Python\;” 兩個目錄，必須連原來的值也都必須打上去。

d:\>path
PATH=C:\Windows\system32;C:\Windows;C:\Windows\System32\Wbem;C:\Windows\System32\WindowsPowerShell\v1.0\;C:\Windows\System32\OpenSSH\;C:\Git\cmd;C:\Users\Eric\AppData\Local\Microsoft\WindowsApps;

d:\>path C:\Python\Scripts\;C:\Python\;C:\Windows\system32;C:\Windows;C:\Windows\System32\Wbem;C:\Windows\System32\WindowsPowerShell\v1.0\;C:\Windows\System32\OpenSSH\;C:\Git\cmd;C:\Users\Eric\AppData\Local\Microsoft\WindowsApps;

d:\>path
PATH=C:\Python\Scripts\;C:\Python\;C:\Windows\system32;C:\Windows;C:\Windows\System32\Wbem;C:\Windows\System32\WindowsPowerShell\v1.0\;C:\Windows\System32\OpenSSH\;C:\Git\cmd;C:\Users\Eric\AppData\Local\Microsoft\WindowsApps;

上面執行了三行命令，先顯示path 環境變數更改前的狀態，然後設定 path 環境變數的內容，最後再顯示path 環境變數更改後的狀態，則可看到 “C:\Python\Scripts\;C:\Python\;” 已經加到前面了。

使用環境變數

前面說過，Path是環境變數之一，所以用 set 顯示全部環境變數時，path 也在其中，而 path 跟 set path= 是一樣的。

而且 Path 最常做的動作，就是增加一個路徑到既有 Path變數，老是要打滿或是copy paste這麼多路徑，未免不太實際，所以要利用 %符號 來使用環境變數了，用%符號加在環境變數前後，CMD在執行時，自動會將環境變數內容替換掉。 看下面例子，第一行先顯示舊的Path變數，第二行就是將 c:\python 加在既有 Path變數的最前面，第三行再看一次Path變數更動過後的內容，可以看到，已經成功地將 c:\python 加到最前面了，而且是利用Path變數，不是真的靠打字打了那麼多路徑。

C:\Users\Eric>path
PATH=C:\Python\Scripts\;C:\Python\;C:\Windows\system32;C:\Windows;C:\Windows\System32\Wbem;C:\Windows\System32\WindowsPowerShell\v1.0\;C:\Windows\System32\OpenSSH\;C:\Git\cmd;C:\Users\Eric\AppData\Local\Microsoft\WindowsApps;

C:\Users\Eric>path c:\python;%PATH%

C:\Users\Eric>path
PATH=c:\python;C:\Python\Scripts\;C:\Python\;C:\Windows\system32;C:\Windows;C:\Windows\System32\Wbem;C:\Windows\System32\WindowsPowerShell\v1.0\;C:\Windows\System32\OpenSSH\;C:\Git\cmd;C:\Users\Eric\AppData\Local\Microsoft\WindowsApps;
